SIEM
SECURITY INFORMATION & EVENT MANAGEMENT
SIEM is the central nervous system of your SOC. It collects, normalises and correlates log data from every source — firewalls, endpoints, cloud, identity — and surfaces anomalies and threats in real time.

It answers: "What is happening across my infrastructure?"
Log Aggregation — Collect logs from 100s of sources into one place
Correlation Engine — Match events across sources to find attack patterns
Real-Time Alerting — Fire alerts the moment thresholds are breached
Dashboards & Reports — Visibility across the entire estate
Compliance — PCI-DSS, HIPAA, ISO 27001 audit trails
⚡
SOAR
SECURITY ORCHESTRATION, AUTOMATION & RESPONSE
SOAR is the action engine that acts on SIEM intelligence. It automates repetitive analyst tasks, orchestrates multi-tool response workflows, and closes incidents with machine speed.

It answers: "What do we do about it — automatically?"
Playbook Automation — Predefined workflows execute on alert trigger
Tool Orchestration — API-connect NGFW, EDR, IAM, ticketing systems
Threat Enrichment — Auto-query VirusTotal, Shodan, MISP, threat intel
Case Management — Track incidents from detection to closure
MTTR Reduction — Mean time to respond drops from hours to seconds

// 04 — Real-world examples
Threat Scenarios in Action
🔐
Brute Force / Credential Stuffing Attack
Target: Azure AD, VPN Gateway, OWA — Common enterprise entry points
SIEM + SOAR
Scenario: An attacker uses a credential dump from a dark web forum and launches 5,000 login attempts against your organisation's Azure AD / Okta tenant and VPN gateway in under 10 minutes. On attempt 4,987, they succeed with a valid credential.
// SIEM Detection + SOAR Response Workflow
SIEM
Log Ingestion: Azure AD sign-in logs, Palo Alto GlobalProtect VPN logs, and Okta event stream are ingested into Microsoft Sentinel or Splunk in real time.
SIEM
Correlation Rule Fires: Rule triggers: ">100 failed logins to the same account within 5 minutes from a single IP → High Severity Alert." Alert: Credential Stuffing Detected.
SOAR
Auto Enrichment: SOAR (Palo Alto XSOAR / Splunk SOAR) instantly queries VirusTotal, Shodan, and internal threat intel feeds on the source IP — malicious reputation confirmed.
SOAR
Automated Containment: Source IP blocked on Palo Alto NGFW via API call. Azure AD Conditional Access forces step-up MFA on the compromised account. Zscaler ZPA session for the user is terminated.
SOAR
Ticket & Notify: ServiceNow incident ticket auto-created with full context attached. SOC analyst receives Slack / Teams alert with one-click Approve / Escalate / False Positive buttons.
✅ Result: Attack detected in <2 min. Containment fully automated in <90 seconds. Analyst reviews a pre-filled closed ticket. Zero manual firewall rule edits or email chains required.
💀
Ransomware Lateral Movement
Post-exploitation: encryption spreading across finance VLAN via SMB
SIEM + SOAR
Scenario: A ransomware binary executes on an endpoint in the finance segment. The malware begins encrypting files, traversing SMB shares, and attempting Pass-the-Hash lateral movement to 47 other hosts on the same VLAN.
// SIEM Detection + SOAR Response Workflow
SIEM
EDR Correlation: CrowdStrike / Microsoft Defender for Endpoint logs ingested. SIEM correlates: rapid file rename events + SMB enumeration + LSASS memory access = Ransomware Behaviour Pattern match.
SIEM
Network Flow Correlation: SIEM joins EDR alert with VMware NSX-T east-west traffic logs — confirms anomalous SMB traffic spike from the infected host to 47 peers in the same subnet.
SOAR
Host Isolation: SOAR calls CrowdStrike Falcon API — network isolation applied to infected host within 30 seconds. NSX-T micro-segmentation quarantine tag applied automatically, cutting all lateral traffic.
SOAR
Forensics Preservation: Azure VM snapshot taken for forensic imaging. Ransomware binary hash submitted to VirusTotal and internal sandbox (Any.run / Cuckoo). IOCs extracted and pushed to threat intel platform.
SOAR
Incident War Room: PagerDuty critical alert fires. Dedicated Slack incident channel auto-created with all enriched context, runbook link, and IR team tagged automatically.
✅ Result: Blast radius contained to 1 host. Lateral movement to 47 peers stopped. MTTR reduced from 2–4 hours of manual response to ~4 minutes of automated containment.
🕵️
Insider Threat — Data Exfiltration
Departing employee exfiltrating patient records to personal cloud storage
SIEM Heavy
Scenario: A healthcare employee on the HR termination list begins downloading 4.2 GB of ePHI patient records from a restricted SharePoint library at 11 PM Friday night, then uploads them to a personal Dropbox account — behaviour completely outside their normal baseline.
// SIEM Detection + SOAR Response Workflow
SIEM
UEBA Behavioural Baseline: Microsoft Sentinel / Exabeam UEBA flags: login outside normal hours + 4.2 GB outbound to Dropbox via ZIA proxy + access to 3 restricted ePHI folders = Risk Score 97/100. Critical alert fired.
SIEM
DLP + Identity Correlation: Zscaler ZIA DLP alert (PHI data pattern match) correlated with Active Directory HR attribute — SIEM confirms user is on the active termination list, elevating severity to Critical.
SOAR
Guardrailed Account Disable: SOAR playbook queries HRMS API to confirm termination status as a guardrail before acting. Confirmed → Zscaler ZPA session revoked, Azure AD account disabled, Microsoft 365 session tokens invalidated.
SOAR
Evidence & Compliance: Full session logs, ZIA DLP capture, and SharePoint audit logs exported to immutable Azure Blob storage. Legal team and HIPAA Compliance Officer auto-notified per Breach Notification playbook with 60-day breach clock started.
✅ Result: Potential HIPAA breach detected and stopped in under 5 minutes at 11 PM with zero analyst on-call involvement. Full evidence chain preserved for legal action. Compliance SLA met automatically.
🎣
Phishing Campaign → Business Email Compromise
Spear-phishing with malicious macro targeting finance team → C2 callback
SOAR Heavy
Scenario: A spear-phishing email with a weaponised Excel macro is delivered to 14 finance team members. Three open it. One executes the macro which establishes an HTTPS-based C2 channel to a command-and-control server in Eastern Europe.
// SIEM Detection + SOAR Response Workflow
SIEM
Email Security Correlation: Microsoft Defender for O365 / Proofpoint alert ingested. SIEM correlates delivery across 14 mailboxes — same sender hash, same attachment SHA256, same X-Originating-IP. Campaign identified.
SOAR
Mass Email Retraction: SOAR playbook calls Microsoft Graph API — offending email purged from all 14 inboxes, Deleted Items, and Sent folders in under 60 seconds. No user awareness required.
SOAR
IOC Propagation: Sender domain, sending IP, and attachment hash automatically pushed to Palo Alto NGFW External Dynamic List (EDL), Zscaler ZIA block list, and Fortinet FortiGuard threat intel within 3 minutes.
SIEM
C2 Beaconing Detected: SIEM detects periodic HTTPS beaconing (every 30s) from the machine that executed the macro to a newly-registered domain. High-severity incident auto-created — SOAR playbook triggered again for host isolation.
SOAR
User Outreach & EDR Scan: Automated phishing awareness email sent to the 3 openers. All 14 recipients' machines queued for CrowdStrike deep EDR scan. Results auto-attached to incident ticket.
✅ Result: Campaign contained org-wide in <5 minutes. 11 users never saw the email. C2 channel blocked before any data exfiltration. Full IOC intel shared across all security controls simultaneously.
☁️
Cloud Misconfiguration — Public S3 Bucket Exposure
AWS S3 bucket with PII made publicly accessible by developer misconfiguration
SIEM + SOAR
Scenario: A developer accidentally disables Block Public Access on an S3 bucket holding 80,000 customer records. Within 8 minutes, an automated cloud scanner begins enumerating and downloading data from the exposed bucket.
// SIEM Detection + SOAR Response Workflow
SIEM
CloudTrail + CSPM Correlation: AWS CloudTrail log ingested into Splunk / Sentinel. SIEM correlates: PutBucketPublicAccessBlock API with BlockPublicPolicy=false + subsequent GetObject calls from external IPs = Critical Cloud Exposure alert.
SIEM
Unusual Access Volume: SIEM detects 1,400 GetObject API calls in 6 minutes from a non-corporate IP range — data exfiltration in progress. Alert escalated to Critical.
SOAR
Automated Remediation: SOAR calls AWS Boto3 SDK via Lambda — Block Public Access re-enabled on the S3 bucket within 45 seconds. Bucket ACL reset to private. External access cut immediately.
SOAR
Scope Assessment: SOAR enumerates all objects accessed by the external IP from CloudTrail logs — 1,247 records confirmed exposed. CloudWatch alarm set for any future public access changes on production buckets.
SOAR
Compliance Breach Workflow: GDPR 72-hour notification clock triggered automatically. DPO, Legal, and Engineering leads notified with full impact report. Jira ticket created for developer remediation and post-incident review.
✅ Result: Exposure window: 8 minutes. Automated fix applied in 45 seconds after detection. Breach scope determined without manual log review. GDPR compliance workflow initiated automatically.
