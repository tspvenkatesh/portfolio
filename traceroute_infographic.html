🛣️ Understanding Traceroute

Network Path Analysis & Route Tracing

Complete Guide to Diagnosing Network Hops and Connectivity Issues
What is Traceroute?
1
Definition
🗺️
Route Visualization Tool
Traceroute is a network diagnostic tool that displays the route (path) taken by data packets as they travel from source to destination across the Internet.
Shows each hop (router) along the way with response times and enables identification of where connectivity breaks.
2
Purpose
🎯
Path Diagnostics
Traceroute maps the network path to identify routing issues, slow connections, network congestion, and hop-specific problems.
Essential for advanced network troubleshooting when PING alone isn't sufficient.
3
Mechanism
⏱️
TTL-Based Discovery
Uses incrementing TTL (Time To Live) values to elicit ICMP responses from each router, building a map of the path layer by layer.
Each hop returns a response, allowing visualization of the entire network path.

Traceroute Variants & Commands

Windows (Tracert)

# Basic tracert
tracert 8.8.8.8

# Specify maximum hops
tracert -h 15 8.8.8.8

# Set timeout (milliseconds)
tracert -w 3000 8.8.8.8

# No DNS resolution
tracert -d 8.8.8.8

# IPv6 tracert
tracert -6 2001:4860:4860::8888

Linux/Mac (Traceroute)

# Basic traceroute
traceroute 8.8.8.8

# ICMP mode (instead of UDP)
traceroute -I 8.8.8.8

# Specify maximum hops
traceroute -m 15 8.8.8.8

# Set timeout (seconds)
traceroute -w 2 8.8.8.8

# No DNS resolution
traceroute -n 8.8.8.8

# Use TCP port 80
traceroute -T -p 80 8.8.8.8

Advanced: MTR (My Traceroute)

# Basic MTR (real-time monitoring)
mtr 8.8.8.8

# Report mode (non-interactive)
mtr --report 8.8.8.8

# TCP mode
mtr --tcp 8.8.8.8

# Specific number of packets
mtr --report-cycles 100 8.8.8.8
Understanding Traceroute Output

Typical Traceroute Output Example

traceroute to google.com (142.251.41.14), 30 hops max

1 192.168.1.1 (192.168.1.1) 1.234 ms 1.156 ms 1.289 ms
2 10.0.0.1 (10.0.0.1) 3.456 ms 3.123 ms 3.567 ms
3 203.0.113.45 (203.0.113.45) 8.234 ms 8.456 ms 8.123 ms
4 * * * (timeout)
5 198.51.100.1 (ae1-0.rt1.sjc.example.net) 12.345 ms 12.567 ms 12.234 ms
6 209.85.169.41 (209.85.169.41) 14.567 ms 14.234 ms 14.456 ms
7 142.251.41.14 (google.com) 15.234 ms 15.456 ms 15.123 ms
Analysis:
  • Hop 1: Your local gateway (very low latency ~1ms)
  • Hop 2: ISP gateway (slightly higher ~3ms)
  • Hop 3: Internet backbone (increasing ~8ms)
  • Hop 4: Timeout (*) - Router not responding to traceroute, but not blocking - packet likely continues
  • Hop 5-6: Cross-network routing (10-14ms)
  • Hop 7: Destination reached (15ms total)
Interpreting Traceroute Results
Result Meaning Possible Cause Action
Successful Response Router responding with RTT Normal hop in path Monitor RTT for anomalies
* * * (Timeout) No response from router ICMP blocked or router misconfigured Usually harmless; continue trace
Request Timed Out Stuck at specific hop Router down, network congestion, or firewall Investigate that specific hop
High RTT Jump Sudden increase in latency Long distance, overloaded router, or congestion Check network usage, consider alternate routes
Increasing RTT per Hop Normal latency accumulation Expected behavior over distance Normal; indicates healthy path
Decreasing TTL TTL approaching 0 Too many hops to destination Increase TTL or check for routing loops
Same IP Multiple Hops Load balancing Different paths through same network Expected in large networks
Geographic Backtrack Packet returns toward source Multi-path routing or network inefficiency Acceptable but suboptimal routing

💡 Tips & Best Practices
Firewall ICMP Blocks
Many routers don't respond to traceroute probes. Seeing "*" is normal. Trace will continue if firewall allows IP-level traffic.
Different Protocols
Linux default is UDP, Windows uses ICMP. UDP can bypass some firewalls. Try ICMP mode (-I flag) if UDP doesn't work.
Run Multiple Traces
Run traceroute multiple times. Path may vary due to load balancing or routing changes. Compare results.
Check Documentation
Cross-reference hops with router hostnames. Many ISPs publish router locations and names publicly.
Typical Hop Count
Most Internet paths: 10-20 hops. More than 30 suggests possible routing issue. Less than 5 indicates local network.
Share with ISP
When reporting ISP issues, always include traceroute output. Shows exactly where problem occurs in their network.
